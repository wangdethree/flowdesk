## 表结构设计

### Ticket 工单主表

核心字段：

| 字段 | 作用 |
| --- | --- |
| `title` | 工单标题，列表页和搜索常用 |
| `description` | 工单详细描述 |
| `category` | 工单分类，例如故障、需求、咨询 |
| `priority` | 优先级，例如低、中、高、紧急 |
| `status` | 状态，例如待处理、处理中、已解决、已关闭 |
| `creator` | 创建人，指向用户表 |
| `assignee` | 处理人，指向用户表，可以为空 |
| `due_at` | 截止时间，用于后续超时提醒 |
| `resolved_at` | 解决时间 |
| `closed_at` | 关闭时间 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### TicketComment 工单记录表

核心字段：

| 字段 | 作用 |
| --- | --- |
| `ticket` | 所属工单 |
| `author` | 记录作者 |
| `content` | 评论或处理内容 |
| `comment_type` | 记录类型：普通评论或处理记录 |
| `created_at` | 创建时间 |

## 为什么要拆成两张表？

工单主表只保存工单当前状态和核心信息。评论、补充说明、处理过程会不断增加，如果全部塞进工单主表，就会导致字段混乱，也无法保留完整处理历史。

拆成 `Ticket` 和 `TicketComment` 后，一个工单可以对应多条记录，这就是典型的一对多关系。

## 为什么要用外键？

外键用来表达表之间的关系：

- `Ticket.creator` 表示这张工单是谁创建的。
- `Ticket.assignee` 表示这张工单分配给谁处理。
- `TicketComment.ticket` 表示这条记录属于哪张工单。
- `TicketComment.author` 表示这条记录是谁写的。

外键的好处是关系清晰，查询方便，也能让数据库帮助维护数据一致性。

## 为什么要加索引？

后续工单列表接口经常会出现这些查询：

- 按状态筛选：只看待处理工单。
- 按优先级排序：紧急工单排前面。
- 按创建人查询：用户只看自己创建的工单。
- 按处理人查询：处理人只看分配给自己的工单。
- 按创建时间排序：最新工单排前面。

所以我们先给这些高频查询组合加索引：

```python
models.Index(fields=['status', 'priority', '-created_at'])
models.Index(fields=['creator', '-created_at'])
models.Index(fields=['assignee', '-created_at'])
```

索引可以提升查询速度，但不是越多越好，因为索引也会占空间，并且会让写入和更新变慢。

## 核心概念

### 什么是 ORM？

ORM 是对象关系映射。简单说，就是用 Python 类表示数据库表，用对象表示表里的记录。Django 的 model 就是 ORM 的核心。

### 什么是迁移？

迁移是数据库结构变更记录。我们改了 model 后，通过 `makemigrations` 生成迁移文件，再通过 `migrate` 把这些变化应用到数据库。

### 什么是一对多关系？

一对多表示一条主记录可以对应多条子记录。比如一张工单可以有多条评论，所以 `Ticket` 到 `TicketComment` 是一对多。

### `CASCADE` 和 `SET_NULL` 有什么区别？

`CASCADE` 表示主记录删除时，相关子记录也一起删除。  
`SET_NULL` 表示主记录删除时，外键字段置空。

在本项目里：

- 工单创建人删除时，工单也删除，使用 `CASCADE`。
- 工单处理人删除时，工单保留，只把处理人置空，使用 `SET_NULL`。

## 面试题

1. Django model 和数据库表是什么关系？
2. 什么是数据库迁移？
3. 外键有什么作用？
4. 一对多关系怎么在 Django 里表示？
5. 为什么工单和评论要拆成两张表？
6. 什么情况下应该给字段加索引？
7. 索引有什么缺点？
8. `CASCADE` 和 `SET_NULL` 有什么区别？

## 简短回答

1. 一个 model 通常对应一张数据库表，字段对应表字段。
2. 迁移是数据库结构变更记录，用来同步 model 和数据库。
3. 外键用来表达表之间的关联，并维护数据关系。
4. 在多的一方使用 `ForeignKey` 指向一的一方。
5. 因为一张工单会有多条评论和处理记录，拆表可以保留完整历史。
6. 高频查询、筛选、排序字段适合加索引。
7. 索引会占用空间，并降低写入和更新速度。
8. `CASCADE` 会级联删除，`SET_NULL` 会把外键置空。
